# 个人 Markdown 笔记软件需求说明

## 1. 文档信息

- 项目名称：LinkNotes（暂定名）
- 文档类型：需求说明
- 文档版本：v0.1
- 编写日期：2026-03-23
- 目标平台：Windows 桌面端优先，Android 为后续扩展

## 2. 项目背景

本项目面向个人使用场景，目标是开发一款本地优先的 Markdown 笔记软件。软件强调以下能力：

- 以本地文件夹作为笔记库，避免用户被专有格式绑定
- 提供文件树、编辑、预览三栏工作界面
- 支持 Markdown 原生输入与快捷命令输入共生
- 提供底部终端面板，满足桌面工作流需求
- 保持界面简洁、稳定、适合长时间写作

项目第一阶段重点是完成 Windows 桌面端可用版本，先建立稳定的本地写作体验，再考虑移动端和同步能力。

## 3. 产品目标

### 3.1 核心目标

- 让用户能够在本地文件夹中高效创建、组织和编辑 Markdown 笔记
- 提供所见即所得程度较高的实时预览体验
- 将常用 Markdown 语法通过快捷插入降低输入成本
- 为高级用户提供与当前工作区联动的内置终端

### 3.2 设计原则

- 本地优先：笔记内容直接落盘为标准 Markdown 文件
- 简洁优先：第一版不引入复杂协作和云端能力
- 兼容优先：保存结果必须保持标准 Markdown 语义
- 可扩展优先：架构上为后续标签、双链、索引和同步预留空间

## 4. 目标用户

- 个人知识管理用户
- 偏好本地文件存储的 Markdown 用户
- 需要轻量写作与整理工具的开发者、学生、研究者
- 希望在笔记环境中直接调用终端的桌面用户

## 5. 使用场景

- 在本地文件夹中维护长期笔记
- 边写 Markdown 边查看实时预览效果
- 通过快捷命令快速插入标题、代码块、待办项等结构
- 在当前笔记目录下打开终端执行脚本、导出、Git 命令
- 对已有笔记进行搜索、整理、重命名、移动和归档

## 7. 功能需求

### 7.1 工作区管理

#### 7.1.1 打开工作区

- 用户可以选择一个本地文件夹作为工作区
- 工作区中的 Markdown 文件和目录应在左侧文件树中展示
- 系统应记住最近打开的工作区

#### 7.1.2 工作区切换

- 用户可以切换到其他工作区
- 切换前若存在未保存变更，系统应先完成自动保存或给予提示

### 7.2 文件树管理

#### 7.2.1 文件树展示

- 左侧展示目录树、Markdown 文件和附件目录
- 支持目录展开和折叠
- 支持高亮当前选中文件

#### 7.2.2 文件操作

- 新建笔记
- 新建文件夹
- 重命名笔记或文件夹
- 删除笔记或文件夹
- 移动笔记到其他文件夹

#### 7.2.3 最近与收藏

- 支持显示最近打开文件
- 支持收藏常用笔记

### 7.3 编辑器

#### 7.3.1 基础编辑

- 中间区域提供 Markdown 编辑器
- 支持多行编辑、撤销重做、复制粘贴、快捷键
- 支持基本 Markdown 高亮显示
- 支持显示当前文档字数、光标位置、保存状态

#### 7.3.2 自动保存

- 用户编辑时应自动保存
- 自动保存应尽可能避免阻塞输入
- 顶部或状态栏应显示保存状态

#### 7.3.3 图片拖拽插入

- 支持将图片拖入编辑器
- 系统自动将图片复制或移动到工作区附件目录
- 编辑器自动插入 Markdown 图片语法

### 7.4 Markdown 预览

#### 7.4.1 实时预览

- 右侧区域实时渲染当前 Markdown 内容
- 支持标题、列表、代码块、引用、表格、链接、图片等常见语法

#### 7.4.2 同步滚动

- 编辑区和预览区应支持同步滚动
- 用户可在设置中关闭同步滚动

#### 7.4.3 预览区域控制

- 预览区域支持显示/隐藏
- 当窗口宽度不足时，可自动折叠预览区域

### 7.5 快捷插入命令

#### 7.5.1 设计原则

- 快捷插入仅作为编辑辅助，不引入新的持久化语法
- 展开后保存到文件中的内容必须为标准 Markdown

#### 7.5.2 初始命令集

- `/h1` 展开为 `# `
- `/h2` 展开为 `## `
- `/h3` 展开为 `### `
- `/dmk` 或 `/code` 展开为代码块模板
- `/todo` 展开为 `- [ ] `
- `/quote` 展开为 `> `
- `/link` 展开为链接模板
- `/img` 展开为图片模板
- `/table` 展开为表格模板

#### 7.5.3 触发方式

- 输入命令后按 `Tab`
- 后续可扩展为输入 `/` 时弹出命令候选面板

#### 7.5.4 共生原则

- 用户直接输入 `## 标题` 时，系统应正常识别为二级标题
- 用户输入 `/h2` 后，也可展开为 `## `
- 两种方式最终效果一致

### 7.6 搜索

#### 7.6.1 全局搜索

- 支持按文件名搜索
- 支持按文件内容搜索
- 搜索结果应显示匹配文件及命中摘要

#### 7.6.2 搜索入口

- 顶部工具栏提供搜索入口
- 支持快捷键快速聚焦搜索框

### 7.7 导出

#### 7.7.1 HTML 导出

- 当前文档支持导出为 HTML

#### 7.7.2 PDF 导出

- 当前文档支持导出为 PDF

### 7.8 内置终端

#### 7.8.1 终端位置

- 终端为底部抽屉式面板
- 默认隐藏，可通过按钮或快捷键唤出

#### 7.8.2 终端能力

- 支持在当前工作区或当前笔记目录中打开终端
- 支持 PowerShell 或 pwsh
- 支持基本输入输出显示

#### 7.8.3 使用边界

- 内置终端仅在桌面端提供
- Android 端不提供系统级终端能力

### 7.9 设置

- 主题切换
- 字体和字号设置
- 预览开关
- 同步滚动开关
- 快捷命令说明
- 默认终端设置

## 8. 非功能需求

### 8.1 性能

- 打开中小型工作区时应保持可接受响应速度
- 编辑器输入延迟应足够低，不影响连续输入
- 预览刷新应控制在用户可接受范围内，避免明显卡顿

### 8.2 稳定性

- 自动保存机制需尽量减少数据丢失风险
- 文件操作需避免误覆盖和异常删除

### 8.3 可维护性

- 模块边界清晰
- 编辑器、预览、文件树、终端分层明确
- 预留后续移动端和索引扩展空间

### 8.4 兼容性

- 文件内容必须保持标准 Markdown
- 附件存储方式应对普通文件系统友好

## 9. MVP 定义

第一版最小可用产品包含以下内容：

1. 打开本地工作区
2. 左侧文件树
3. 新建、重命名、删除笔记
4. Markdown 编辑器
5. 实时预览
6. 自动保存
7. 基础搜索
8. 图片拖拽插入
9. 快捷插入命令
10. 导出 HTML/PDF
11. 底部终端面板

## 10. 验收标准

### 10.1 基础可用性

- 用户能在工作区中创建并编辑 Markdown 文件
- 用户能在编辑时实时看到预览结果
- 用户关闭应用后再次打开，文件内容保持正确

### 10.2 快捷命令

- `/h2` 能正确展开为 `## `
- `/dmk` 能正确展开为代码块模板
- 展开后的内容在其他 Markdown 编辑器中可正常识别

### 10.3 终端

- 用户可通过按钮或快捷键打开底部终端
- 终端工作目录正确指向工作区或当前文件目录

### 10.4 导出

- 用户可以将当前文档导出为 HTML
- 用户可以将当前文档导出为 PDF
